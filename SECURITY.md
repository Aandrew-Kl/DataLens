# Security Policy

## Supported Versions

The latest release of DataLens is supported with security updates. Earlier releases are not supported.

| Version | Supported |
| ------- | --------- |
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

If you believe you have found a security vulnerability in DataLens, please report it privately and do not open a public issue.

You can report vulnerabilities by:

- Emailing `security@example.com`
- Using GitHub Security Advisories for this repository

Please include, where possible:

- A clear description of the issue and its potential impact
- Steps to reproduce the issue
- Any proof-of-concept, logs, or screenshots that help validate the report
- Your contact information for follow-up questions

## What to Expect

After you submit a report, maintainers will:

- Acknowledge receipt within 48 hours
- Investigate and validate the report
- Provide status updates as appropriate during triage and remediation
- Target a fix or mitigation within 30 days for confirmed vulnerabilities

Response and remediation timelines may vary depending on severity, complexity, and release coordination requirements.

## Scope

### In Scope

The following areas are in scope for responsible disclosure:

- Authentication and authorization issues
- Data handling, storage, or exposure vulnerabilities
- API security issues, including access control and input validation problems

### Out of Scope

The following are generally out of scope:

- Denial of service attacks or resource exhaustion issues
- Social engineering, phishing, or attacks requiring human manipulation
- Reports that depend on unsupported or outdated releases
