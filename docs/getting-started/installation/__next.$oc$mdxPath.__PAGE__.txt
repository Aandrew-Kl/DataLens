# Installation

## Recommended: Docker Compose

For most contributors and self-hosted deployments, Docker Compose is the quickest way to get everything running.

```bash
cp .env.example .env
docker-compose up
```

If you use the newer Docker CLI plugin, the equivalent command is:

```bash
docker compose up --build
```

Services started by the repository compose file:

- Frontend on `http://localhost:3000`
- Backend on `http://localhost:8000`
- PostgreSQL on `localhost:5432`

## Manual Installation

Use the manual path when you want to work on the frontend and backend independently.

### Requirements

- Node.js `22+`
- Python `3.12+`
- PostgreSQL `16+`
- npm `10+`

### Frontend

From the repository root:

```bash
npm install
cp .env.example .env.local
npm run dev
```

### Backend

From `backend/`:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
alembic upgrade head
uvicorn app.main:app --reload --port 8000
```

## Required Environment Variables

These are the three values you should verify first:

- `JWT_SECRET`: Signs bearer tokens for login and protected API access.
- `DATABASE_URL`: Points the backend at PostgreSQL before migrations and API requests run.
- `OLLAMA_URL`: Points AI-assisted flows at your local or network-reachable Ollama instance.

The sample repository defaults also define `NEXT_PUBLIC_API_URL`, `NEXT_PUBLIC_WS_URL`, and `OLLAMA_MODEL`.

## Troubleshooting

### `docker-compose up` fails because ports are already in use

Stop the local services bound to `3000`, `8000`, or `5432`, or override ports in your compose override file.

### The backend cannot connect to PostgreSQL

Check that `DATABASE_URL` matches the running host, port, user, password, and database name. For local development, run migrations again after fixing the connection string.

### The frontend loads but API requests fail

Confirm `NEXT_PUBLIC_API_URL` points at the FastAPI server and that CORS still allows `http://localhost:3000`.

### AI actions hang or fall back unexpectedly

Verify that Ollama is running and that `OLLAMA_URL` resolves from the process that is making the request.
