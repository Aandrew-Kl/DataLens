# Error tracking with Sentry

Sentry support is optional. The app only initializes Sentry when a DSN is set, so leaving the Sentry environment variables empty keeps telemetry disabled.

## Create a Sentry project

1. Sign in to Sentry and create a project for this application.
2. Copy the DSN from the project settings.
3. Add the DSN to your local environment or deployment platform.

## Configure environment variables

Set at least one of these variables:

- `NEXT_PUBLIC_SENTRY_DSN` for the browser runtime
- `SENTRY_DSN` for the server and edge runtimes

You can also set:

- `NEXT_PUBLIC_SENTRY_ENVIRONMENT` to label events by environment
- `NEXT_PUBLIC_SENTRY_TRACES_SAMPLE_RATE` to control tracing volume

If `SENTRY_DSN` is not set, the server and edge runtimes fall back to `NEXT_PUBLIC_SENTRY_DSN`.

## Privacy defaults

Personally identifiable information is not sent by default. Request cookies and common auth and cookie headers are scrubbed before events leave the app.
