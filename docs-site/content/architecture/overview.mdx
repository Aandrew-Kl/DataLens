# Architecture Overview

DataLens uses a hybrid architecture:

- DuckDB-WASM in the browser for local query execution and profiling
- Next.js for the interactive frontend shell
- FastAPI for auth, persistence, analytics, and AI endpoints
- PostgreSQL for durable backend state

## Why This Design

This split gives DataLens its two main properties:

- **Fast:** exploratory queries, previews, and many chart workflows happen locally in the browser without a round trip for every SQL statement.
- **Private:** the core analytics loop can stay on the user’s machine, while backend services are used only when you need persistence, auth, collaboration-adjacent workflows, or server-side ML and analytics endpoints.

## High-Level Flow

```text
CSV / JSON / Excel
        |
        v
Browser parsers -> DuckDB-WASM -> query results -> charts / dashboards / reports
        |                                 |
        |                                 v
        +------> Next.js frontend <-------+
                     |
                     v
             FastAPI application
             |       |        |
             |       |        +--> Ollama (optional local AI)
             |       |
             |       +--> ML / analytics endpoints
             |
             +--> PostgreSQL for users, datasets, history, and metadata
```

## Practical Result

The frontend stays responsive for exploratory work, while the backend remains focused on the jobs that benefit from a durable service boundary.
